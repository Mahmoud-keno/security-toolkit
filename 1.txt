
1. SDES (Simplified DES)

Reference:
Textbook "Cryptography and Network Security" - Chapter 3: Block Ciphers

Purpose & Educational Value:

* Designed specifically for teaching cryptography concepts
* Demonstrates all principles of DES without complexity
* Perfect for understanding Feistel networks

Technical Specifications:

* Key Size: 10 bits (vs 56 in real DES)
* Block Size: 8 bits (vs 64 in real DES)
* Rounds: 2 rounds (vs 16 in real DES)
* Subkeys: 2 subkeys of 8 bits each

Algorithm Steps:

1. Key Generation:
   Initial 10-bit key → P10 permutation → Split → Left shifts → P8 → K1
   → More shifts → P8 → K2

2. Encryption:
   Plaintext → Initial Permutation (IP) → Complex Function (FK) using K1
   → Swap → FK using K2 → Inverse IP → Ciphertext

3. Complex Function (FK):

* Expansion/Permutation (4→8 bits)
* XOR with subkey
* S-box substitution (8→4 bits)
* Permutation (P4)

Strengths for Learning:

1. Simplicity: Easy to trace through each step
2. Complete: Contains all DES components in miniature
3. Hands-on: Can be done manually with pen and paper
4. Foundation: Perfect prep for understanding real DES

Limitations:

1. Not Secure: Only 2¹⁰ = 1024 possible keys
2. Too Small: 8-bit blocks vulnerable to analysis
3. Educational Only: Never used in real systems

Why It's Important:

* First exposure to Feistel structure
* Teaches substitution-permutation networks
* Shows how multiple rounds increase security
* Demonstrates key scheduling

2. DES (Data Encryption Standard)

Reference:
FIPS PUB 46-3 (1977), NIST Standard for 20+ years

Historical Significance:

* 1972: NIST requests encryption standard proposals
* 1974: IBM submits "Lucifer" cipher (128-bit key)
* 1975: NSA modifies design (56-bit key, S-box changes)
* 1977: Published as FIPS 46
* 1999: Broken by brute force (DES Challenge III)

Technical Architecture:

A. Key Schedule Generation (Detailed):

Step 1: Start with 64-bit key (8 bytes with parity bits)
Example: 0x133457799BBCDFF1

Step 2: Apply PC-1 (64→56 bits)

* Removes parity bits (8th, 16th, 24th, ..., 64th)
* Permutes remaining 56 bits

Step 3: Split into C0 (28 bits) and D0 (28 bits)

Step 4: For rounds 1-16:

* Left circular shift Ci, Di (1 or 2 bits based on round)
* Combine CiDi (56 bits)
* Apply PC-2 to get Ki (48 bits)

B. Encryption Process (Feistel Network):

1. Initial Permutation (IP): 64-bit plaintext → permuted
2. Split into L0 (32 bits) and R0 (32 bits)
3. 16 Rounds:
   a. Li = Ri-1
   b. Ri = Li-1 ⊕ F(Ri-1, Ki)
4. Final: Combine R16L16 → Inverse IP → Ciphertext

C. The F-Function (Heart of DES):

1. Expansion: 32-bit R → 48-bit using E-table
2. XOR: with 48-bit round key Ki
3. S-box Substitution: 48-bit → 32-bit (8 S-boxes, 6→4 bits each)
4. Permutation: 32-bit → 32-bit using P-table

S-Box Design (Critical Security Feature):

* 8 different S-boxes, each 4×16
* Carefully designed by NSA to resist cryptanalysis
* Each S-box has specific properties:

  * No linear or affine approximations
  * Output not correlated to input
  * Changing 1 input bit changes ≥ 2 output bits

Strengths (Why It Lasted 20+ Years):

1. Strong Design: Withstood public scrutiny for decades
2. Hardware Efficient: Easy to implement in hardware (1 chip)
3. Thorough Analysis: Extensively studied, few weaknesses found
4. Foundation: Inspired AES and other modern ciphers

Weaknesses & Attacks:

1. Key Size (56 bits): Main weakness

* 2⁵⁶ ≈ 72 quadrillion keys
* 1998: EFF's Deep Crack (250,000 dollar machine, 56 hours)
* 1999: Distributed.net (22 hours)

2. Brute Force Vulnerabilities:

* 1977: Estimated 2,000 years to break
* 1999: 22 hours with distributed computing
* Today: Minutes with modern hardware

3. Theoretical Attacks:

* Differential Cryptanalysis (discovered 1990)
* Linear Cryptanalysis (discovered 1993)
* Both require 2⁴⁷ chosen plaintexts (impractical but theoretical)

Modes of Operation (Critical for Real Use):

1. ECB (Electronic Codebook): Weak - identical blocks give identical ciphertext
2. CBC (Cipher Block Chaining): Adds IV, prevents patterns
3. CFB (Cipher Feedback): Turns block cipher into stream cipher
4. OFB (Output Feedback): Similar to CFB, error propagation differences

Triple DES (3DES): The DES Successor

3DES Encryption:
C = EK3(DK2(EK1(P)))

3DES Decryption:
P = DK1(EK2(DK3(C)))

* Key options: 2-key (112-bit) or 3-key (168-bit)
* Slower: 3× slower than DES
* Secure: Still used in some legacy systems
* Standard: FIPS 46-3 (withdrawn 2023)

Educational Value Today:

1. Teaches: Block cipher fundamentals
2. Shows: Importance of key size
3. Demonstrates: Feistel network advantages
4. Historical: First major public cipher standard

Current Status:

* Not Recommended for new systems
* Legacy Use only in 3DES form until 2023
* Educational perfect for learning cryptography
